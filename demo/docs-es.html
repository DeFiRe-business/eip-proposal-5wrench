Coercion-Resistant Vault

ERC-8238 | Guia paso a paso

Que es esto?

Una smart wallet que protege contra ataques de coercion fisica ("ataques de la llave inglesa de $5"). Divide tus fondos en:

  • Balance caliente (Hot) — cantidad que puedes gastar al instante, con limite diario (ej: 0.05 ETH/dia).
  • Boveda fria (Cold) — la mayor parte de tus fondos, bloqueados con timelock (5 min a 72h) y/o aprobacion de guardianes.

Bajo coercion: "Solo puedo enviarte esta cantidad. El resto esta bloqueado y no puedo acceder mas rapido." Verificable on-chain.

Arquitectura

Requisitos previos

1MetaMask — instala desde metamask.io.

2Red Sepolia — en MetaMask, cambia a Sepolia test network (activa "Mostrar redes de prueba" en configuracion si no aparece).

3Sepolia ETH — gratis en sepoliafaucet.com o cloud.google.com/application/web3/faucet/ethereum/sepolia.

Conectarse

4Abre la demo y haz clic en "Connect MetaMask". Cambia a Sepolia automaticamente.

5El banner indica si eres el vault owner (acceso completo) o visitante (solo depositar).

Dashboard

  • Total balance — todo el ETH en el contrato.
  • Hot (naranja) — lo que puedes gastar ahora, limitado por el spending limit.
  • Cold (azul) — parte bloqueada, requiere timelock o guardianes.
  • Spent this epoch — hot budget consumido en las ultimas 24h.

Acciones

Depositar ETH

Cualquiera puede depositar. Introduce cantidad, clic en Deposit, confirma en MetaMask. Se actualiza tras minar (~12s).

Hot spend

Transferencia instantanea dentro de tu presupuesto hot.

  1. Introduce direccion + cantidad.
  2. Clic en Hot Spend, confirma.
  3. Transferencia instantanea. "Spent this epoch" sube.
Si excedes el budget, revierte con "ExceedsHotBudget". Se resetea cada 24h.

Cold withdrawal

  1. Introduce destinatario + cantidad, clic en Request Withdrawal.
  2. Aparece entrada con cuenta atras.
  3. Al llegar a cero, clic en Execute.
Cualquier guardian puede cancelar antes de la ejecucion — el boton de panico.

Swap en Uniswap V3

Intercambia ETH por USDC con contratos reales de Uniswap en Sepolia.

  1. Introduce cantidad, clic en Wrap + Swap.
  2. Confirma 3 transacciones: wrap ETH, aprobar router, swap.
  3. El vault tiene USDC. Contador hot en cero.
Clave: Las operaciones DeFi via execute() son conservadoras de valor — no consumen spending limit. Solo hotSpend() consume budget.

Despliega tu propio vault

Puedes desplegar tu propia instancia en Sepolia en unos pocos pasos. Necesitas Foundry instalado y un RPC Sepolia gratuito de Alchemy o Infura.

1Clonar y configurar

git clone https://github.com/DeFiRe-business/eip-proposal-5wrench
cd eip-proposal-5wrench
cp .env.example .env

Edita .env y rellena SEPOLIA_RPC_URL, DEPLOYER_PRIVATE_KEY (usa una wallet desechable, no tu key principal), VAULT_OWNER (tu direccion de MetaMask) y ETHERSCAN_API_KEY (gratis en etherscan.io).

2Desplegar y verificar

source .env
forge script script/Deploy.s.sol:Deploy \
  --rpc-url $SEPOLIA_RPC_URL \
  --broadcast \
  --verify \
  --etherscan-api-key $ETHERSCAN_API_KEY

El script muestra la direccion del vault desplegado y un enlace a Etherscan. Copia la direccion en demo/deployment.sepolia.json para conectar el frontend a tu vault.

3Fondear y hacer whitelist

Envia Sepolia ETH a la direccion del vault. Luego haz whitelist de WETH y el router de Uniswap (necesario para la demo del swap):

VAULT=0xTuDireccionDelVault

# Programar whitelist (sujeto a timelock)
cast send $VAULT "setWhitelistedTarget(address,bool)" \
  0xfFf9976782d46CC05630D1f6eBAb18b2324d6B14 true \
  --rpc-url $SEPOLIA_RPC_URL --private-key $DEPLOYER_PRIVATE_KEY

cast send $VAULT "setWhitelistedTarget(address,bool)" \
  0x3bFA4769FB09eefC5a80d6E87c3B9C650f7Ae48E true \
  --rpc-url $SEPOLIA_RPC_URL --private-key $DEPLOYER_PRIVATE_KEY

# Esperar a que expire el timelock, luego ejecutar
cast send $VAULT "executeWhitelistChange(address)" \
  0xfFf9976782d46CC05630D1f6eBAb18b2324d6B14 \
  --rpc-url $SEPOLIA_RPC_URL --private-key $DEPLOYER_PRIVATE_KEY

cast send $VAULT "executeWhitelistChange(address)" \
  0x3bFA4769FB09eefC5a80d6E87c3B9C650f7Ae48E \
  --rpc-url $SEPOLIA_RPC_URL --private-key $DEPLOYER_PRIVATE_KEY

4Lanzar la demo

cd demo
python -m http.server 8000

Abre http://localhost:8000 y conecta MetaMask.

Modelo de seguridad

Bajo coercion

El atacante solo puede extraer el hot balance. El resto esta bloqueado con timelock. Verificable on-chain. La cuenta atras da tiempo a los guardianes para cancelar.

Guardianes

Direcciones de confianza que pueden: cancelar withdrawals, aprobar via multisig (bypass timelock), pausar el vault (24h auto-expiry), despausar con multisig.

Configuracion protegida

Subir el spending limit tiene timelock. Solo las reducciones (mas seguras) son inmediatas.

Preguntas frecuentes

Dinero real?

No. Solo testnet Sepolia.

"ExceedsColdBalance"?

Balance total menor que spending limit — todo esta en hot. Deposita mas.

El swap falla?

El pool de Sepolia puede tener poca liquidez. El wrap demuestra el mecanismo.
